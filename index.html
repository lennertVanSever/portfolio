Projects I worked on

So far I have only worked on a project basis. Most projects took around 6 months up to a year to be developed, although a project is never really finished. Every time I worked together with a small team of 2 up to 6 great people, keep that in mind since I don’t want to take all the credit.

Approvals - Borealis Digital Studio

Approvals makes it easier to approve or reject items as a manager within Borealis. The purpose is that it becomes so easy that managers would do it themself and not delegate it to their assistants.

For this project, I worked as a frontend developer to assure that the user interface, experience and integration was coming together. All the components were custom build in ReactJS and state management handled by Redux.

Vintage map - Personal

To keep myself busy during the covid winter lockdown, I made a web application to create your own vintage maps. You can easily export it to a PDF or order a print. Try it here: https://vintagemap.app

The map is built with D3.js for maximum flexibility, the rest of the interface is made with vanilla web components. In April 2021 it went live by featuring on Producthunt.

NutriGuide - Borealis Digital Studio

NutriGuide calculates for farmers/resellers which fertilizers they should use for their plot to get the optimal yield within the legal framework. It’s available in Austria and France, you can even try it yourself.

Just before NutriGuide was released I was assigned to the project to be solely responsible for the frontend. The most challenging part was the customization needs for each role and country without ending up with an endless amount of if statements. The solution was to put all the specific logic in configuration files.

Another challenge was to make the form as easy to use as possible since a lot of parameters have to be filled in to make an accurate calculation. Under the hood, the interface was built with the React Material UI component library. State management was handled with Redux.

Artist connections - Personal

While browsing on Spotify I wondered whether all the artists in the world are “related” to each other. To proof it, I used the Spotify API to find the 20 related artists of my favourite artist Whitney. I iterated this process until all the found artists were covered. Eventually I ended up with 1,212,134 artist and 21,261,611 connections. I used this data as nodes and relationships in a Neo4j graph database.

Afterwards, I created a simple website where you can look up the relation between any two given artists. It’s quite amazing how even the most distinct artists are still related to each other. If you want more detailed information, feel free to read my Medium blog post.

Erasmus+ Digital Tools - European University Foundation

As an Erasmus student, I was annoyed by how inefficient the administration of the Erasmus+ program was. I decided to do something about it with my bachelor thesis: Digitizing the Erasmus+ programme. I contacted the European Commission with my initial idea and they send it through to the European University Foundation who then mentored my bachelor thesis.

Eventually, they decided to hire me so that I could work further on my project. After 2 months we had a proposal to create the Erasmus+ app. The European Commission agreed to fund it and after 8 months it was developed and released to the world. With 40 000 downloads, I was happy with the results but it was only useful for students and not universities so afterwards I build the Erasmus+ dashboard to help them with their mobility administration.

For both applications, I was the only developer so I was responsible for the design, frontend, back end and data management. For the development, I used ReactJS, MeteorJS, Cordova, NodeJS, GraphQL, MongoDB and MySQL.

Technologies I used

I enjoy combining my frontend, backend and UI/UX design skills to create useful web applications. However, most of my time is spend on frontend technologies but I think it’s still essential to learn from the other fields

Frontend development

I started creating my first web applications in 2013 with only HTML and CSS. Back then responsive web design was all the rage so even my first creations had responsive layouts.

Today I primarily use ReactJS with Styled Components and hooks. For state management, I tend to switch between Redux and Apollo local state management. In the future, I would like to learn TypeScript and maybe Flutter. I highly value performance, accessibility, and best practices so I often use Google Lighthouse to score my websites.

Backend development

During my studies, I mostly worked with PHP, Java and MySQL. Now I like to develop with NodeJS since then it requires less context switching when moving between the frontend and backend.

On the database side, I use either MongoDB or PostgreSQL depending on the context. I love to work with GraphQL since I think it’s the future of API design.

UX/UI design

Understanding user needs is critical for any proper project. For this reason, I highly value user interviews and feedback. Whenever I’m building a web application, I always keep in mind how the user would perceive it.

For user interface design I often use Sketch and Abstract for version control. However, I only use it to get a rough idea since you are way closer to the real interaction when designing in the browser.

Books I read

Development and design is my real passion so I’m more than happy to sacrifice some free time to learn more about it. I only mention the 5 most inspiring books since I tend to read a new book every three weeks.

  • Better web typography, Matej Latin; Web design is 90% typography so we must give it appropriate attention. I learned how to make the ‘perfect’ paragraph and how to get the vertical and horizontal rhythm right through macro/micro typography.
  • You Don’t Know JS series, Kyle Simpson; Truly understanding how JavaScript works can be the difference between a good and a great developer. Thanks to Kyle I’m more confident when writing JavaScript code.
  • Form Design Patterns, Adam Silver; Forms are hard for both developers and users. With the mentioned design patterns I was able to create much simpler forms than before.
  • Clean Code, Robert C. Martin; Because of this book it’s my daily work goal to make my code as explicit as possible.
  • Atomic Design, Brad Frost; After reading this book I started structuring my UI components with Atomic Design in mind which makes a lot of sense in my personal opinion.
